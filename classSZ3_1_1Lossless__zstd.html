Attention When dstCap is smaller than the space needed, ZSTD will not throw any errors. Instead, it will write a portion of the compressed data to dst and stops. This behavior is not desirable in SZ, as we need the whole compressed data for decompression. Therefore, we need to check if the dst buffer (dstCap) is large enough for zstd
